What are environment Variables?
Environment variables are key-value pairs stored in the system that affect how processes behave. Th ane "key" is the name of the variable(usually written in ALL CAPS)and "value" is data stored inside it.

Think of them like: "Global settings" that programs can read.

Environment variables are used in web apps, databases, devops, containers, apis. without them modern software development is almost impossible.

eg 	USER=pratham
	HOME=/home/pratham
	PATH=/usr/bin:/bin:/usr/local/bin

USER--> your username of linux
HOME--> ur home directory, path to current user home dir
PATH--> where system looks for commands




======= Why are environment variables imporatant? ========
They help:
- Programs find files and executables
- Store configuration (like database URL,API Keys)
- Customize user environment

eg when u type ls command, Linux searches for ls inside directories listed in PATH

========= Types of Variables ==========
1. Local Variables - Only available in current shell and temporary , Not passed to child processes 
eg name ="Honey"
   echo $name

2. Environment Variables = Available system-wide (or session-wide), Passed to child processes
eg export name="Honey"


======= Managing Variables (The "How-to")=========
Managing variables depends on how long you want them to last.

Temporary Variables(Shell session only)
if you define a variable directly in the terminal, it disappears the moment you close that terminal window.

MY_PROJ="Secret Weapon"
echo $MY_PROJ

Exporting Variables -
By default, a variable you create isn't shared with "child" processses (like a script you run). To share it, you use export:

export API_KEY="3857984"

Permanent Variables(the Config Files)
To make a variable stick around even after a reboot, you add it to your shell's configuration file (usually .bashrc  or .zshrc in your home folder).
1. Open the file: nano ~/.bashrc
2. Add your line at the bottom:
	export TOOL_PATH="/opt/tools"
3. Save and referesh: source ~/.bashrc


======== Simple example: The "greeting" script==========
Let's say you are building a script that needs to know your favorite color to customize the output.

1. Set the variable
	export FAV_COLOR="Orange"

2. Create a bash script (greet.sh):
	echo "Hello $USER, I see your favorite color is $FAV_COLOR."
3. Run it:
The script will pull $USER from the system and $FAV_COLOR from what you just defined. If you change FAV_COLOR="Red" and run it again, the script adapts without you having to change the code.


=================== $PATH Variable ==============
The most imporatant environment variable is $PATH. When you type ls or python, the system doesn't know where those programs live. 
It looks through the folders listed in $PATH one by one until it finds them. If you install a new tool and the terminal says "command not found" it usually means you need to add that tool's folder to your $PATH.
