💽 2. What is a Storage Device in Linux?

A storage device = anything that stores data:

Hard disk
SSD
USB drive
Cloud volume

In Linux:

Devices are treated as files

📁 3. Device Files (Core Concept)

All devices are represented in:

/dev

Examples:

/dev/sda → first disk
/dev/sdb → second disk
/dev/sda1 → partition 1
🧠 Think:
Physical Disk → Linux maps it → File in /dev
🔍 4. Types of Storage Devices
📌 Block Devices
Store data in blocks
Example: disks

👉 /dev/sda

📌 Character Devices
Stream data
Example: keyboard

👉 For storage → we mainly deal with block devices

🧱 5. Disk Structure (Important)

A disk is divided into:

Disk
Partitions
File systems
🧩 Step-by-step:
Disk → Partition → File System → Mount → Use
📦 6. Partitioning (VERY IMPORTANT)
❓ Problem:

One big disk → how to organize?

👉 Solution: Divide into partitions

Example:
/dev/sda
 ├── /dev/sda1
 ├── /dev/sda2

Each partition acts like separate disk.

Tools:
fdisk
parted
📁 7. File System Creation

After partition → it’s empty.

👉 Need to format:

mkfs.ext4 /dev/sda1
Why?
Creates structure to store files
🔗 8. Mounting (Connecting Storage)

Partition is useless until mounted.

mount /dev/sda1 /mnt/data

👉 Now you can access it

AWS Mapping:
Attach disk using Amazon EBS
Use it on Amazon EC2
Then mount it
🧠 9. Logical Volume Manager (LVM) — MOST IMPORTANT

Now comes real DevOps-level concept

❓ Problem (Without LVM)
Disk size fixed
Partition size fixed
Hard to resize
💡 Solution: LVM

LVM gives flexible storage

🧩 LVM Layers:
Disk → PV → VG → LV → File System
📌 1. Physical Volume (PV)
Actual disk or partition
pvcreate /dev/sda1
📌 2. Volume Group (VG)
Pool of storage
vgcreate my_vg /dev/sda1
📌 3. Logical Volume (LV)
Virtual partition
lvcreate -L 10G -n my_lv my_vg

👉 Then format + mount

🔥 Why LVM is powerful:
Resize storage easily
Combine multiple disks
Take snapshots
AWS Analogy:
LVM = combining multiple EBS volumes
Flexible scaling like cloud storage
📊 10. Swap Space (Memory Extension)
❓ Problem:

RAM is full → system crashes?

💡 Solution: Swap
Disk used as RAM backup
Commands:
mkswap /dev/sda2
swapon /dev/sda2

👉 AWS analogy:

Like using disk when RAM is exhausted in EC2
