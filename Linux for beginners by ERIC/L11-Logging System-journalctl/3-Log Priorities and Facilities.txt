Log Priorities and Facilities

Logging and logfiles reflect the different priorities and facilities of the
operating system and the system administrator. Some logs are very
important and are given a priority of 0 or emerg(ency), while others are less
important and are given priorities of 6 (info) or 7 (debug). 

The system administrator can determine which logs have which priority. In addition, the
system administrator can select which priority logs they want to view.

------------------------------------------------------------
Here is a list of the logging system priorities:
emerg (0)
alert (1)
crit (2)
err (3)
warning (4)
notice (5)
info (6)
debug (7)
------------------------------------------------------------


To view the highest-priority logfiles, we can construct a journalctl query
using the -p option, such as:
journalctl –p "emerg"

$ journalctl -p "emerg"
-- No entries --

========== How to view logs for custom service==================

The journalctl utility also enables us to view the logs of a specific service
using the -u option. To view the logs associated with the apache2 web
server, we can construct a query such as:

kali> journalctl –u apache2
May 28 14:25:48 kali systemd[1]: Starting apache2
May 28 14:25:48 kali apachectl[118241]: AH00558:
May 28 14:25:48 kali systemd[1]: Started apache2
May 29 00:00:31 kali systemd[1]: Reloading apache
--snip--
This query can be used to find the logs of any service in Linux simply by
appending the -u option followed by the name of the service.
