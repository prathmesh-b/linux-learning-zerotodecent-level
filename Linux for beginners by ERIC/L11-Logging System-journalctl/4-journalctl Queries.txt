journalctl Queries

One of the major advantages of journalctl is the ability to query the logs
specifically for events, users, and time, as journalctl has a query language
that is very similar to normal human language.

For instance, if I wanted to
see only the events in the last 24 hours, I can query:

kali> journalctl –q –-since "24 hours ago"

================= how to serach events logs of USER like root or other ====================
The journalctl utility also enables you to search for events by user. 

As you
know, the root user in Linux is designated with the user ID of 1000. So, we
can search for events linked to the root user by building the journalctl query
like this:

kali> journalctl _UID=1000 –-since "24 hours ago"

============= what if kernel is having problem how to check ========================
The kernel of any operating system is its heart. It controls the entire
operating system, and it is what makes Linux, Linux. If the kernel is having
problems, we probably want to be aware of that. To view kernel logs, we
can use the -k option (kernel). For kernel logs within the last 24 hours, we
can enter:
kali> journalctl –k -–since "24 hours ago"
