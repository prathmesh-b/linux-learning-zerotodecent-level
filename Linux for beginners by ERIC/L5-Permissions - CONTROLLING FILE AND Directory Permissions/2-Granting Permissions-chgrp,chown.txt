Granting Permissions

root user can grants level of permissions to the file depending on what they need to do with file.


Each and every file must be allocated a particular permission level for the different identities using it. 

The three levels of permission are as follows:
     r  -   Permission to read grants permission only to open and view a file.
     w  -   Permission to write allows users to view and edit a file.
     x  -   Permission to execute allows users to execute a file (but not
necessarily view or edit it).

r - able to open and view/read a file
w - make edit to file
x - to execute file but not necessarily view or edit

============ 1. Granting Ownership to an Individual User/Owner ================

To move ownership of a file to a different user so that they have the ability
to control permissions, we can use the chown (or change owner) 

------------
-- chown ---
------------

command: chown bob/tmp/bobsfile
Here, we give the command, the name of the user we are giving ownership
to, and then the location and name of the relevant file. This command grants
the user account for Bob ownership of bobsfile.

chown - change owner
there is file and want to chane owner of that file.

The chown command in Unix-like operating systems is used to change the user and/or group ownership of files and directories. This is crucial for managing access control and security on a multi-user system. 

Syntax

chown [OPTIONS] OWNER[:GROUP] FILE(s)

chown: The base command.
[OPTIONS]: Optional flags that modify the command's behavior (e.g., -R for recursive changes).
OWNER: The username or numeric user ID (UID) of the new owner.
[:GROUP]: An optional group name or ID (GID), preceded by a colon, to also change the group ownership simultaneously.
FILE(s): The name of one or more target files or directories. 

Note: In most cases, changing the owner of a file requires sudo or root privileges, as regular users cannot give away ownership of their files for security reasons. 

Common Examples
Change only the owner of a file:

chown john filename.txt
This changes the owner of filename.txt to the user john, leaving the group unchanged.

============= 2.Granting Ownership to a Group ============
To transfer ownership of a file from one group to another, we can use the
chgrp (or change group) command.

------------
-- chgrp ---
------------

In organisation when different groups/team working on project clients. You need some group have root permission and some have less permissions.

root group install new software and they need to give to seniordesign group for their work.
Let’s say the root group downloads and installs a design program named
newIDSdesign; they will then need to transfer ownership to the senior design group name seniordesign so
the senior design group can use it at will. To do so, 

the root group can enter the following command:
 sudo chgrp seniordesign newIDSdesign
 
 sudo chgrp grpnamewhichinherit programname/file
This command passes the seniordesign group ownership of newIDSdesign software.

The next set of permissions (r--) represents those of the group and shows
that the group has read permission but not write or execute permissions.
And, finally, we see that the rest of the users also have only read permission
(r--).

These permissions aren’t set in stone. As a root user or file owner, you can
change them. Next, we’ll do just that.

Changing Permissions
We can use the Linux command chmod (or change mode) to change the
permissions. Only a root user or the file’s owner can change permissions.
