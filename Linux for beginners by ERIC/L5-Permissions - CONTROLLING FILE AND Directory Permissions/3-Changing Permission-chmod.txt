Changing Permissions

We can use the Linux command chmod (or change mode) to change the
permissions. Only a root user or the file’s owner can change permissions.

we use chmod to change permissions of file using
two different methods. 
1-  use a numerical representation of permissions, and then we 
2- use a symbolic representation.

------------
-- chmod ---
------------


The chmod command in Linux (short for change mode) is used to modify the access permissions of files and directories. 
Permissions determine who can read, write, or execute a file, 
and are assigned to three classes of users: the file's owner (u), members of the file's group (g), and others (o). 

syntax
chmod ___ filename/dir

Syntax
The basic syntax is chmod [options] mode file_name. You can specify the mode using either symbolic (letters and symbols) or numeric (octal numbers) notation. 

----------------------------------------------------------------------------------------------------------

================================ Changing Permissions with UGO ==============================

Although the numeric method is probably the most common method for
changing permissions in Linux, some people find chmod’s symbolic method
more intuitive. Both methods work equally well, so choose the one that
suits you. The symbolic method is often referred to as UGO syntax, which
stands for 

UGO =   user (or owner), group, and others.


====== Permissions and User Classes =====

Class 	Symbol	Description
User	u	The owner of the file.
Group	g	Users who are members of the file's group.
Others	o	All other users not in the above categories.
All	a	All users (u, g, and o).

u - owner of file
g - group
o - guest or all other users

Permission Symbol	Octal Value	Description
Read	     r	            4	                  Allows viewing file contents or listing directory contents.
Write	     w	             2	                 Allows modifying/deleting a file or creating/deleting files in a directory.
Execute	     x	              1	                 Allows running an executable file or entering (searching) a directory.

You can also change multiple permissions with just one command. If you
want to give both the user and other users (not including the group) the
execute permission, you could enter the following:

kali> sudo chmod u+x, o+x hashcat.hcstat

This command tells Linux to add the execute permission for the user as well
as the execute permission for others for the hashcat.hcstat file.

================================ Changing Permissions with Decimal Notation ==============================


We can use a shortcut to refer to permissions by using a single number to
represent one rwx set of permissions. Like everything underlying the
operating system, permissions are represented in binary, so ON and OFF
switches are represented by 1 and 0, respectively. You can think of the rwx
permissions as three ON/OFF switches, so when all permissions are
granted, this equates to 111 in binary.

We can then easily represent a binary set like this as a single digit by
converting it into octal, an eight-digit number system that starts with 0 and
ends with 7. An octal digit represents a set of three binary digits, meaning
we can represent an entire rwx set with one digit.

r-read w-write x-execute

Binary Octal rwx 
000      0   ---
001      1   --x
010      2   -w-
011      3   -wx
100      4   r--  to set only read permission to file 
101      5   r-x
110      6   rw-
111      7   rwx

are on, they are represented by the octal equivalent of 7.
